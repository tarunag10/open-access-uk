# Product Map

Open Access UK is a static, browser-only civic toolkit for public-service access, evidence, escalation, and open contribution.

## Personas

- Citizens use the tools to draft requests, understand routes, prepare evidence, and keep local copies.
- Advisers use the tools to prepare plain-English packs with a client before sending or escalating.
- Service designers use the form and design-system tools to improve accessible public-service journeys.
- Contributors use the public repos and maintainer helper to find focused, beginner-safe improvements.
- Maintainers use source metadata, repo metadata, and release checks to keep high-risk content reviewable.

## Core Journeys

### Ask For Information Or An Adjustment

1. Start in the umbrella site workflow.
2. Open the letter generator or legal templates.
3. Draft locally in the browser.
4. Copy, print, download, or create a local action pack.
5. Use the public service directory if the reply is delayed or refused.

### Prepare A Complaint Or Escalation

1. Search the public service directory by issue or sector.
2. Build a readiness report and evidence checklist.
3. Keep a contact log locally.
4. Use a legal template or letter follow-up where suitable.
5. Check source notes and official routes before submitting.

### Improve A Public-Service Form

1. Open Accessible Public Forms.
2. Review a form pattern and readiness score.
3. Copy a remediation report, JSON spec, or local action pack.
4. Use the design system for matching tokens, copy patterns, and components.
5. Verify keyboard, screen-reader, contrast, and non-digital fallback paths.

### Contribute To The Suite

1. Start from the umbrella site or parent README.
2. Pick a public repo or contributor-tooling task.
3. Use the maintainer helper to create issue text, onboarding packs, and action plans.
4. Run the repo `npm test` and `npm run build` commands.
5. Open a focused pull request with accessibility, privacy, and source notes.

## Tool-To-Tool Flows

- Letter generator -> public service directory: follow up or escalate after a missed response.
- Legal templates -> public service directory: pair a template with the correct complaint or ombudsman route.
- Accessible forms -> design system: move from a form audit to reusable accessible patterns.
- Maintainer helper -> parent metadata: turn source/repo health into maintenance tasks.
- Umbrella site -> every tool: route users by goal without accounts or tracking.

## Backlog Themes

- Metadata-driven navigation and feature registry.
- Source and safety visibility for high-risk content.
- Privacy centre and local data controls.
- Shared export, deadline, route, and taxonomy utilities.
- FOI, SAR, complaint follow-up, guide cards, glossary, and deadline planner.
